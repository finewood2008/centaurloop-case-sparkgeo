# Roadmap

This roadmap describes the intended path from case prototype to production-ready CentaurLoop application.

## Phase 1: Case Prototype

Status: complete enough for local demonstration.

- Content growth loop configuration
- Brand memory onboarding
- AI planning and draft generation
- Human review gates
- Manual publishing workspace
- Published URL feedback analysis
- Cycle review and memory confirmation
- Built-in experience runtime and OpenAI-compatible runtime adapter

## Phase 2: Backend Foundation

- Move local Vite API routes to a deployable backend
- Add authentication and workspace isolation
- Persist cycles, tasks, feedback, and memories server-side
- Add rate limits and request validation for crawling routes
- Store model configuration securely

## Phase 3: Better Feedback Intelligence

- Add platform-specific feedback adapters
- Support authenticated user-provided platform sessions where legally and technically appropriate
- Detect public counters more reliably
- Compare published content against original drafts
- Add feedback confidence scores

## Phase 4: More CentaurLoop Cases

- Sales follow-up loop
- Customer success loop
- Hiring/onboarding loop
- Product research loop
- Finance reporting loop

## Phase 5: Production Operations

- Observability and event logs
- Audit trail for human approvals
- Team collaboration
- Role-based permissions
- Scheduled loop execution
- Notification channels

# 路线图

这份路线图描述了 SparkGEO 从案例原型走向可生产化 CentaurLoop 应用的路径。

## 阶段 1：案例原型

状态：已经足够支撑本地演示。

- 内容增长闭环配置
- 品牌记忆启动引导
- AI 规划和草稿生成
- 人工审核卡点
- 人工发布工作台
- 发布 URL 反馈分析
- 周期复盘和记忆确认
- 内置体验运行时与 OpenAI-compatible runtime adapter

## 阶段 2：后端基础

- 将本地 Vite API 路由迁移到可部署后端
- 增加登录鉴权和工作区隔离
- 服务端持久化 cycles、tasks、feedback、memories
- 为抓取路由增加限流和请求校验
- 安全存储模型配置

## 阶段 3：更强的反馈智能

- 增加平台专属反馈适配器
- 在合法且技术可行的前提下支持用户授权的平台 session
- 更稳定地识别公开计数器
- 对比发布内容和原始草稿
- 增加反馈置信度

## 阶段 4：更多 CentaurLoop 案例

- 销售跟进闭环

- 财务报告闭环

- 可观测性和事件日志
